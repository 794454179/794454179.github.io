Java：构造方法、this修饰符、访问权限修饰符、多态、抽象、static关键字、常用类

一、知识点总结

1、构造方法

（1）作用：专门用来创建对象的，并且可以初始化属性的值。

（2）如果代码中，没有构造方法，编译器会自动添加一个无参的构造。

（3）如果代码中，写了构造方法，编译器都不会再添加无参的构造方法。

（4）编码习惯：写了有参构造，再写上无参构造。

（5）无返回值（也不能写void）；方法名必须和类名一致；修饰符最多只能访问权限为public；

（6）每当创建一个对象，就会调用一次，执行1次。

2、this修饰符

2.1 this用法一

（1）词义：表示当前对象。（谁调用谁就是当前的对象 ）

（2）一个类中，成员变量和局部变量名字可以一样。但遵循就近原则。二者都有并使用时离哪个声明近就是哪个变量。

（3）使用this，就是告诉编译器，this后面的变量是成员变量。

2.2 this用法二

（1）this()指代当前类的构造方法。

3、访问权限修饰符（public、private……）

​ private：私有的。本类中。

​ 默认的（default）：啥修饰符都不写，就是默认。本类，同包下可以访问。

​ protected：受保护的。本类，同包下，不同包只有其子类可用。

​ public：没有限制。

4、多态、向上转型、向下转型、instanceof

4.1 对象的向上转型(其实就是【多态】 写法)

4.1.1 格式

​ 父类名称 对象名 = new 子类名称();

4.1.2 含义

​ 右侧创建一个子类对象，把它当做父类来看待使用。

4.1.3 注意事项

​ 向上转型一定是安全的。从小范围转向了大范围。

4.1.4 举例

​ Animal animal =new Cat();

​ 创建了一只猫，当作动物看待。

​ 从小范围的猫，向上转型成为更大范围的动物。

4.2 对象的向下转型（其实就是【还原】的动作）

4.2.1 格式

​ 子类名称 对象名 =（子类名称） 对象父类；

4.2.2 含义

​ 将父类对象，【还原】成为本来的子类对象。

4.2.3 注意事项

​ （1）必须保证对象本来创建的时候，就是猫，才能向下转型成为猫。

​ （2）如果对象创建的时候本来不是猫，现在非要向下转型成为猫，就会报错。

4.2.4 举例

​ Animal animal =new Cat(); //本来是猫，向上转型为动物。

​ Cat cat = (Cat)animal; //本来是猫，已经被当作动物了，还原回来成为本来的猫

4.5 聊聊多态性

多态性：事物的多种形态

​ 方法：方法的多种形态，指的就是方法的重载。一个功能方法，但是根据参数的不同，执行的具体方法也不同。（也叫做静态多态。程序编译时发生的多态）

​ 对象：对象的多态性，结合方法的重写。一个子类对象，如果看成父类的类型，那么执行的方法就只能父类的以及子类重写的（向上转型）；如果再把该对象向下转型为子类类型，才可以执行子类的新增方法（向下转型）。（整个也叫做动态多态。）

7、常用类

（Object类、String类、Math类、bigdecimal(精确值)类、随机数类、日期类）

7.1 Object类

所有类的根基，Java中的所有的类，都会直接或间接的继承Object类。

只要是java中的类对象，默认都可以调用Object类中的方法。

  • 如果一个方法的参数为Object类型，实际可以传入任意的类对象。
  • 如果一个方法的返回值是Object类型，实际上可以返回任意的类对象。

（1）getClass()–>Class，未完待续—>反射

​ 返回一个对象运行时的类类型。

​ 对象.getClass()—>返回该对象所属的类的字节码文件对象。

​ 理解为：返回的是堆中的实际的对象的类型

（2）hashCode()—>int，获取对象的哈希码

​ 每当产生一个对象的时候，都是由哈希算法计算出该对象的哈希值。

（3）toString()—>String，返回一个对象的字符串表示形式

​ 对象.toString()—>String，描述这个对象的。

​ Object类：对于toString()–>获取的是对象的内存地址(逻辑地址)

eclipse快捷键：shift + alt + s，可以自动生成toString()。

idea快捷键：alt + insert

（4）equals()–>boolean，用于比较两个对象是否相等。（实际上是比较了对象的内存地址）
  • ==：专门比较数值，是否相等。适合8个基本数据类型：byte，short，int，long，float，double，char，boolean
  • equals()，在Object类中，提供了一个方法equals()，专门用于判断对象是否相等（比较对象的内存地址）。

11.2 TCP协议和UDP协议

（1）TCP协议：重点掌握——>好比打电话：

  • 面向连接，数据传递的时候很安全。
    • 三次握手，四次挥手。
    • 因为面向连接，所以传递的效率较低。
    • 面向连接，数据安全的。可以传递大量的数据。
  • 服务端和客户端
    • 服务端：提供服务的
    • 客户端：来访问服务端，获取数据
      • 客户端登录，数据下载，文件传输。。

（2）UDP协议：了解——>好比写信

  • 面向无连接，数据传的时候不是很安全。
    • 因为面向无连接，效率很高。
    • UDP
  • 没有客户端和服务端的概念。叫做发送方，另一个叫接收方。
